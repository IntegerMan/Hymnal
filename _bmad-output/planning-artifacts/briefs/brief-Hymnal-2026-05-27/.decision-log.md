# Decision Log — Hymnal Product Brief

_Canonical audit trail for all decisions, changes, and overrides made during the brief session._

---

## 2026-05-27 — Discovery Session

**Editor: no preview panel.** Author wants pure markdown editor with syntax highlighting only, not a split rendered preview. Keeps scope minimal and matches writing style preference.

**Gantt: multi-bar rows, manual dates.** Each chapter row shows one box per phase (draft/edit/polish/review) with distinct standard colors. Dates are manually entered; setting a chapter status *may* pre-populate dates but all dates remain editable. No git-derived dates.

**Word count: observed metric with optional targets.** Word count is live-counted from file content, not manually entered. Targets are optional and can be set at chapter, part, or book level. Book has a range target (min/max). Authors may want per-chapter targets too.

**AI: configurable provider.** No bundled AI key. Configurable via LiteLLM, MEAI (Microsoft Extensions for AI), or Microsoft Agent Framework. Author brings their own key/service.

**AI features: late V1.** Core writing and tracking must be solid first. AI (summaries, chat, editorial analysis, issues list) is a V1 feature but deprioritized behind the writing and project management layer.

**Scope: single book / folder-as-workspace.** One open project at a time. Opens a folder, reads `Book.txt`. No multi-project library in V1.

**Target platform: .NET, Windows + Linux.** No macOS required. Cross-platform via .NET (likely Avalonia UI or similar).

**License: MIT open source.** Product name: Hymnal. Author's book: *A Choir of Minds*.

**Out of scope confirmed:** rendered preview, multi-book, collaboration, built-in Git, LeanPub API, mobile/web.

## 2026-05-27 — Review Feedback

**Vision dialed back.** Brief originally framed Hymnal as "the reference environment for Markua authors." Corrected to: personal tool that is open source — not trying to be a platform. Vision rewritten accordingly.

**Success criteria updated.** Added explicit criterion: complete authoring workflow (write, track, AI review, Git commit/push) requires no context switch to another application. Git itself remains external in V1; the rest is self-contained.

**Issues model elaborated.** Issues carry: id, type (enum), description, state (open/resolved/dismissed), created date, optional location (book/part/chapter/paragraph/line). Visual treatment: line-level issues show as margin indicators; chapter/part-level issues show as scope-top summary indicators. Full detail captured in addendum.md for PRD consumption.
