# Addendum: Hymnal Product Brief

_Depth that belongs in downstream documents (PRD, architecture) or earned a place but does not fit the brief's 1-2 page constraint._

---

## AI Issues Model (detail for PRD)

Issues generated by AI analysis carry the following fields:

| Field | Notes |
|---|---|
| `id` | Unique identifier |
| `type` | Enum: readability, grammar, inconsistency, developmental, line-editing, etc. |
| `description` | Human-readable finding from the AI |
| `state` | Enum: open, resolved, dismissed |
| `created_date` | When the issue was created |
| `location` | Optional. Granularity: book, part, chapter, paragraph, or line |

**Visual treatment in the editor:**
- Line-level issues: margin indicator on the relevant line(s)
- Chapter-level issues: summary indicator at the top of the chapter
- Part-level issues: summary indicator at the top of the part scope

Issues are navigable via an issues panel organized by scope (book → part → chapter).

---

## AI Summary Model (detail for PRD)

Summaries are generated and stored at three levels:
- Chapter summary
- Part summary
- Book summary

Summaries are used as context when the author opens an AI chat session, scoped to the current chapter, part, or whole book. Summaries should be regenerable on demand and stored as sidecar files alongside the manuscript (or in a project metadata folder).

---

## Workflow Context (author's stated goal)

The intended complete authoring workflow stays within Hymnal or an adjacent Git client:
- Write prose in Hymnal's editor
- Track chapter status and phase progress in Hymnal
- Run AI editorial queries in Hymnal
- Commit and push progress via Git (external Git client or terminal — Git operations are not built into Hymnal V1)

No spreadsheet, no external project tracker, no context switch to another writing tool.

---

## Word Count Targets (detail for PRD)

- Word count is a live-observed metric, not manually entered
- Targets are optional at any level: chapter, part, book
- Book-level target is a range (min / max word count)
- Per-chapter and per-part targets may be a single value or a range
- Progress indicators in the Gantt and sidebar should reflect target proximity when targets are set

---

## Rejected Alternatives

- **Rendered preview panel**: Explicitly rejected. Author prefers pure markdown editing with syntax highlighting. Adds implementation complexity without matching the authoring style.
- **Git operations built into Hymnal V1**: Explicitly out of scope. Author uses Git externally. May revisit in a future version.
- **Multi-book project library**: Out of scope for V1. One folder = one open project.
- **Collaboration / multi-author**: Out of scope entirely for V1.
