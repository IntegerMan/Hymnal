## Session: 2026-05-27 — Initial PRD Discovery & Draft

### DL-1: Platform targets
**Decision:** Windows 10+ and Linux (Ubuntu LTS, Fedora current) as first-class targets. macOS not targeted for V1.
**Source:** Author (Matthew-Hope) — confirmed during Discovery.
**Captured in:** NFR-P1.

### DL-2: Metadata storage location
**Decision:** `.hymnal-data/` folder at the Workspace root. Git-trackable by default.
**Rationale:** Keeps the Workspace fully portable (copy folder, metadata travels with it). Author preferred keeping data inside the repository.
**Source:** Author — confirmed during Discovery.
**Captured in:** FR-5, NFR-S1.

### DL-3: Corkboard — V1 scope split
**Decision:** Corkboard ships in two stages within V1. Early V1 = status-visualization cards only (FR-28–FR-30). Late V1 = drag-to-reorder, inclusion toggle, and Chapter insertion (FR-31–FR-33).
**Rationale:** The book is already outlined; the immediate value is status visibility, not structural editing.
**Source:** Author — confirmed during Discovery.
**Captured in:** §6.1, §6.2.

### DL-4: Chapter notes — V1
**Decision:** Per-chapter notes (not part of the manuscript) are a V1 Early Core feature.
**Source:** Author — confirmed during Discovery.
**Captured in:** FR-13.

### DL-5: Supplemental docs — V1, folder-based only
**Decision:** Supplemental docs (character bibles, research, etc.) are V1. Structure is author-defined with no enforced schema — just a folder tree within `.hymnal-data/docs/`.
**Source:** Author — confirmed during Discovery. Author explicitly declined structured/linked-entity approach.
**Captured in:** FR-34–FR-37.

### DL-6: Git integration — lightweight V1
**Decision:** Lightweight Git commit/push in V1. The brief previously listed Git as explicitly out of scope; this decision reopens it as a lightweight capability. Scope: stage all → commit (optional message) → push to current remote. No branch management.
**Source:** Author — reversed prior brief decision during Discovery. Brief had explicitly excluded Git ops.
**Captured in:** FR-38–FR-43.

### DL-7: Gantt Phase progress
**Decision:** Optional manually-entered progress percentage (0–100%) per Phase per Chapter. Blank by default.
**Rationale:** Derived-only progress (from Status) was considered but the author was unsure; manual entry gives flexibility without requiring it.
**Source:** Author — confirmed during Discovery ("optionally set one progress per phase").
**Captured in:** FR-24.

### DL-8: Corkboard card face content
**Decision:** Each card shows: Chapter title, Status, Word count, word count percentage toward Target (if Target set), and current-phase start/end dates (if entered).
**Source:** Author — confirmed during Discovery.
**Captured in:** FR-28.

### DL-9: Dark synthwave visual theme
**Decision:** Application-wide dark synthwave theme. Primary accent: purple. Secondary accents: yellow, pink, orange. No light theme for V1. WCAG AA contrast required on text/background pairs.
**Source:** Author — stated during brain dump.
**Captured in:** FR-11, NFR-V1.

### DL-10: Parts roll up chapters in Gantt
**Decision:** Parts appear as collapsible header rows in the Gantt view, rolling up their Chapter rows.
**Source:** Author — stated during brain dump.
**Captured in:** FR-21.

### DL-11: Rendered preview panel — rejected
**Decision:** No rendered Markdown/Markua preview panel. Editor is source-text only.
**Source:** Product Brief (rejected alternatives). Confirmed in scope discussions.
**Captured in:** FR-10, §5 Non-Goals.

### DL-12: Stakes calibration
**Decision:** "Meaningful personal launch" tier. PRD should be thorough enough for AI dev agent and future contributor use, without enterprise ceremony.
**Source:** Facilitator assessment, confirmed by author context (personal + open source, primary user = builder).

## Session: 2026-05-27 — Assumption Resolution Pass

### DL-13: Manuscript folder structure for Parts
**Decision:** Parts are represented as subfolders within `manuscript/`. Each Part folder contains a `part.md` file using `{class: part}` attribute + `# Title` heading (not `# Part N #` syntax). Part folder names are author-defined. `Book.txt` uses folder-prefixed paths for all files (`part1/01-chapter.md`). A subfolder is a Part if and only if it contains `part.md`. Chapter files at `manuscript/` root with no Part subfolder are top-level unparted Chapters.
**Source:** Author — confirmed during assumption resolution. Reference project: `C:\Dev\EliAndGraceMakeAGame`.
**Replaces:** FR-3 assumption (heading-scan approach) and incorrect `# Part N #` heading format.
**Captured in:** FR-3 (updated), Glossary Part entry (updated), addendum manuscript structure diagram.

### DL-14: Word count tokenization approach
**Decision:** Word count uses whitespace tokenization on non-directive, non-attribute-list text. Not a full Markdown AST parse.
**Source:** Author — confirmed (FR-16 assumption resolved).
**Captured in:** FR-16 (assumption tag removed).

### DL-15: Gantt default time axis
**Decision:** The Gantt always shows a time axis. Default span is today through ~one month out. Once Phase dates are entered, the axis adjusts to span earliest–latest Phase date across the Workspace, keeping today visible.
**Source:** Author — confirmed with suggested default during assumption resolution.
**Replaces:** FR-26 assumption (no time axis until dates entered).
**Captured in:** FR-26 (updated).

### DL-16: Chapter move between Parts also moves file on disk
**Decision:** When a Chapter is dragged to a different Part on the Corkboard (Late V1), Hymnal moves the `.md` file into the target Part's subfolder on disk, updates `Book.txt`, and updates sidebar/Gantt grouping. Chapter content is not modified.
**Source:** Author — confirmed FR-31 assumption with addition of `Book.txt` update.
**Captured in:** FR-31 (updated).

### DL-17: Chapter exclusion is authoritative in Book.txt
**Decision:** Excluded chapters are removed from `Book.txt`. Hymnal tracks them in `.hymnal-data/` for re-inclusion. The exact storage format is deferred to implementation.
**Source:** Author — confirmed FR-32 behavior.
**Captured in:** FR-32 (clarified).

### DL-18: AI structured editorial modes in Late V1
**Decision:** Free-text queries are the primary AI chat interaction mode. Three structured modes are targeted for Late V1: **Proofing**, **Consistency**, and **Line Editing**. Each sends a defined prompt and auto-parses results into the Issues list. Additional structured modes are post-V1.
**Source:** Author — partially confirmed FR-47 assumption with specific mode names.
**Captured in:** FR-47 (updated).

## Session: 2026-05-27 — Finalize Pass

### DL-19: OQ-5 resolved — Git push auth failure behavior
**Decision:** On push authentication failure, surface the raw Git error output and leave the local commit in place. No automatic retry, credential prompting, or recovery flow in V1.
**Rationale:** The simpler and safer option. Leaves a clean local commit for the author to push manually once they resolve the auth issue externally.
**Source:** Facilitator (Finalize pass) — OQ-5 self-answered during review.
**Captured in:** OQ-5 (closed), FR-43 (existing behavior matches this decision).

### DL-20: Success Metrics section added
**Decision:** Added §7 Success Metrics with four primary metrics (SM-1–SM-4) and three counter-metrics. Metrics are qualitative — no time-based or numeric thresholds. The AI editorial metric (SM-2) is qualitative by author preference.
**Source:** Facilitator (Finalize pass) — critical structural gap identified in review. Metrics derived from brief success criteria.
**Captured in:** §7 Success Metrics (new section).

### DL-21: FR-12 concrete editor defaults
**Decision:** Focus-first editor defaults: minimum 16px body font size, 1.6 line height, 70ch maximum content width. All adjustable in settings.
**Rationale:** "Comfortable for extended prose writing sessions" was not implementable as a done-ness criterion. These values are standard prose typography best practices.
**Source:** Facilitator (Finalize pass) — FR-12 done-ness gap identified in review.
**Captured in:** FR-12 (updated).

**Decision:** PRD set to `status: final`. Addendum set to `status: final`. Decision log complete through DL-22.
**Source:** Facilitator (Finalize pass complete).
