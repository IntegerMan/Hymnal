## Decisions

**DL-UX-1. Working mode: fast path.**
Author selected fast path. Both spines drafted with `[ASSUMPTION]` tags for gaps not resolvable from sources.

**DL-UX-2. Typography: Inter + JetBrains Mono, embedded.**
Cross-platform constraint (Windows + Linux). Both are free, open-source, and will be embedded in the Avalonia self-contained bundle — no dependency on system fonts. Inter for all UI chrome; JetBrains Mono for the editor. Monospace chosen for the editor because the author sees raw Markua source at all times (no preview panel), and a fixed-width font makes the structural signals (attribute lists, blurb prefixes, heading hierarchy) more legible.

**DL-UX-3. Navigation model: horizontal tab bar in title bar, CORA-lineage.**
Top-center horizontal tabs (WRITE · GANTT · CORKBOARD) with all-caps label convention, derived from CORA (author's prior app). Active tab indicated by a 2px `primary` underline. Secondary icon buttons (AI, Git, Settings) on the right end of the title bar.

**DL-UX-4. Shell layout: three-panel (sidebar · main · right panel).**
Left sidebar (240px, collapsible) + main content region + right panel slot (280px, collapsible). Right panel is a single slot occupied by one of: Notes / AI Chat+Issues / Git. Status bar (24px) persistent at the bottom.

**DL-UX-5. Surface hierarchy: tonal elevation, no shadows.**
Four dark surface tiers: `surface-base` (#0D0B14) → `surface-elevated` (#141020) → `surface-overlay` (#1C1729) → `surface-high` (#241F35). No `box-shadow` used for hierarchy. Borders used only where a structural edge is needed at the same tonal level.

**DL-UX-6. Lifecycle status color mapping.**
Outlining → #6B7480 (gray) · Drafting → #38BDF8 (cyan) · Editing → #9D4EDD (purple) · Polishing → #F5C842 (yellow) · Reviewing → #E91E8C (pink) · Done → #22D3A0 (mint).

**DL-UX-7. Editor font: JetBrains Mono, 16px min, 1.7lh.**
Minimum 16px per PRD FR-12. Line height 1.7 for prose breathing room.

**DL-UX-8. Status bar placement and content.**
Persistent 24px bottom bar: current file path · chapter word count · book word count · status pill (clickable dropdown) · git branch · change count (orange if > 0).

**DL-UX-9. Custom window chrome.**
Avalonia `ExtendClientAreaToDecorationsHint` used to remove native OS title bar and host the custom 40px title bar with integrated tab navigation. Fallback to OS chrome if platform doesn't support it.

**DL-UX-10. Right panel is a single slot.**
Notes, AI Chat + Issues, and Git all share the same right panel position. Only one visible at a time. Prevents visual fragmentation and keeps the editor surface dominant.

**DL-UX-11. Corkboard card anatomy.**
Cards: 180–260px wide (grid-dependent), intrinsic height (minimum ~100px). 4px top status stripe for the lifecycle stage color. Status pill at bottom-left. Drag-to-reorder is Late V1 per PRD FR-31.

**DL-UX-12. AI Chat scope history persists per-scope.**
Switching scope (CHAPTER → PART → BOOK) does not clear the chat history for the previous scope. Each scope maintains its own session history.

**DL-UX-13. Issues panel — flat filter model (resolves PRD OQ-6).**
Issues panel uses flat filters (scope / type / state) rather than a hierarchical tree navigator. Addresses PRD OQ-6.

**DL-UX-14. No custom editor context menu in Early V1.**
Right-click in the editor surface uses the system default (AvaloniaEdit built-in). Exception: clicking an issue margin indicator opens the Issues panel filtered to that issue's scope.
